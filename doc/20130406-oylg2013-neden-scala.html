Tanmlar

Yatay lekleme (Scala out/Scale down)

  • Datk bir uygulamaya yeni bir dm(node) eklemek ya da karmak.

Dikey lekleme (Scale up)

  • Tek bir dme CPU ya da bellek gibi yeni kaynak eklemek.

Parallelism

  • Paralel donanmlar zerinde programlar daha hzl komak.

Concurrency (E zamanllk)

  • paracklarnn e-zamanl olarak komalarn ynetmek.

Concurrency problemi

var x = 0
async { x = x + 1 }
async { x = x * 2 }
// muhtemel sonular 0, 1, 2
  • E-zamanl i paracklarnn paylaml deiken durumlara(mutable shared state) erimesi rasgeleliin sebebidir.

Concurrency problemi

  • non-determinism = paralel ileme + deiken durum

  • Rasgele olmayan ileme istiyorsak deiken durumdan(mutable state) saknmamz gerekli.

Senkronizasyon maliyeti

  • Mutable state varsa e zamanl almann senkronize edilmesi gerekir.
  • Javada aadaki senkronizasyon aralar mevcuttur.
  • Atomik veri yaplar (AtomicInteger, ...)
  • Synchronized bloklar
  • CountDownLatch, CyclicBarrier, Semaphore
  • Eer uygulamanzda mutable state yoksa senkronizasyon gibi bir sorununuz da yoktur.

Threaded vs Evented

  • Threaded: Servlets, Ruby on Rails, Django, PHP

  • Evented: Play, Node.js, Twisted

Threaded sunucular

  • Her bir istee bir thread atarlar
  • Thread cevap gnderilene kadar tm ileri gerekletirirler.
  • Uzak bir servis ars gibi herhangi bir I/O senkrondur.
  • Bu sebeple thread bloklanr ve I/O ilemi tamamlanana kadar bo oturur.

// Apache HttpClient ile example.com'dan bir istek yaplyor.

Snrsz thread oluturmann dezavantajlar

Thread yaam dngs maliyeti

  • Thread oluturma ve ldrme maliyetsiz deildir.
  • Sunucu uygulamalarnda istekler sk ve kk.
  • Her bir istek iin yeni bir thread
  • Yksek kaynak tketimi.

Snrsz thread oluturmann dezavantajlar

Kaynak tketimi

  • Aktif threadler sistem kaynaklarn(zellikle bellek) kullanrlar.
  • Kullanlabilir ilemci saysndan daha fazla thread var ise threadler bo dururlar.
  • Bo duran threadler gereksiz yere bellek kullanrlar.
  • lemciler iin yaran ok sayda thread var ise bu durum ayrca performans kaybna sebep olabilir.

Snrsz thread oluturmann dezavantajlar

Kararllk

  • Oluturulabilecek thread says snrldr.
  • Bu snr stack size gibi parametrelere ve iletim sisteminin snrlarna baldr.
  • Bu snra ulatnzda OOM gibi bir yant alrsnz.
  • Bu durumu dzeltmeye almak risklidir.
  • Bunun yerine uygulamanz bu snra ulamayacak ekilde tasarlamanz daha kolaydr.

Evented sunucular

  • Evented sunucular genelde CPU ekirdei bana yalnzca bir thread kullanrlar.
  • Tm olay threadlerin asla bloklanmayacaklar zerine kuruludur. Tm I/O asenkrondur.
  • Bylece thread bo beklemek yerine dier istekleri ileyebilir
  • I/O cevab hazr olduunda tekrar geri dnebilir.

Play Framework ile async rnei

  • Cloud ile birlikte tipik bir uygulama pek ok uzak servis ars yapmak durumunda kalyor.
object ProxyController extends Controller {

  def proxy = Action {
    val responseFuture: Future[Response] = WS.url("http://example.com").get()

    val resultFuture: Future[Result] = responseFuture.map { resp =>
      Status(resp.status)(resp.body).as(resp.ahcResponse.getContentType)
    }

    Async(resultFuture)
  }

}
